# Telco Customer Churn — Business Intelligence & Predictive Analytics

## Overview
This project implements an end-to-end **Business Intelligence and Predictive Analytics pipeline** for customer churn prediction using SQL and Machine Learning.

The system combines:

- PostgreSQL (data warehouse + BI layer)
- SQL feature engineering
- Random Forest classification
- Model predictions written back to the database
- Analytical SQL queries for business insights

The goal is to identify customers with high probability of churn and support data-driven retention decisions.

---

## Architecture

Kaggle Dataset  
→ PostgreSQL (`telco_raw`)  
→ SQL Feature Engineering (`churn_features`)  
→ Random Forest Model  
→ Predictions stored in DB (`predictions`)  
→ BI Views & Analytics Queries

---

## Dataset
**Telco Customer Churn Dataset**  
https://www.kaggle.com/datasets/blastchar/telco-customer-churn

Contains customer demographics, services, contracts, billing data, and churn outcome.

---

## Database Layer (PostgreSQL)

### Tables
- `telco_raw` — raw imported dataset
- `churn_features` — ML-ready feature table
- `predictions` — churn probabilities generated by the model

### Views
- `churn_top_risk` — highest churn-risk customers
- `churn_actions` — suggested retention actions

SQL is used both for **data modeling** and **Business Intelligence analytics**.

---

## Machine Learning Model

Model: **Random Forest Classifier**

Pipeline:
1. Load features from PostgreSQL
2. Train churn prediction model
3. Evaluate performance
4. Store predictions back into database
